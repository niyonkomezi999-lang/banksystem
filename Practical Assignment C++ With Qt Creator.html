Practical Assignment: Bank Account Management System in Qt Creator

Objective

Create a Graphical User Interface (GUI) application for a Bank Account Management System using Qt Creator and C++. The application should demonstrate:

­       Object-Oriented Programming principles

­       Qt's signal/slot mechanism

­       Event-driven programming

­       GUI design with Qt Widgets

 

Part 1: BankAccount Class (Backend)

Requirements:

Create a BankAccount class with:

Private Data Members:

QString depositorName;

int accountNumber;

QString accountType;  // "Savings" or "Current"

double balance;

Public Member Functions:

// Constructor

BankAccount();

 

// Set account details

void createAccount(QString name, int accNum, QString type, double initialBalance);

 

// Transaction functions

bool deposit(double amount);

bool withdraw(double amount);

 

// Getter functions

QString getDepositorName() const;

int getAccountNumber() const;

QString getAccountType() const;

double getBalance() const;

QString getBalanceString() const;  // Returns balance as formatted string

 

// Display function

QString getAccountDetails() const;  // Returns formatted account details

Part 3: Required Functionality

Menu Options (via Buttons):

1. Create Account Button

ü  Opens a dialog for entering:

­       Depositor Name

­       Account Number

­       Account Type (ComboBox: Savings/Current)

­       Initial Deposit Amount

ü  Validates input

ü  Creates the account object

ü  Displays confirmation message

2. Search/Find Account

­       Enter account number in QLineEdit

­       Click "Search" button

­       Display account details in the information area

­       Show error if account not found

3. Deposit Button

­       Enter amount in Amount field

­       Click "Deposit"

­       Validate account exists

­       Validate amount (positive)

­       Update balance

­       Show success message with new balance

­       Update status bar

4. Withdraw Button

­       Enter amount in Amount field

­       Click "Withdraw"

­       Validate account exists

­       Check sufficient balance

­       Update balance if successful

­       Show error if insufficient balance

5. Check Balance Button

­       Shows current balance in a message box

­       Or displays in the details area

6. Display Details Button

­       Show all account information

­       Use formatted output with proper alignment

7. Reset Button

­       Clears all input fields

­       Resets status messages

­       (Keep account data)

8. Exit Button

­       Confirms exit

­       Closes application

 

Part 4: Event Handling (Signals & Slots)

Required Connections:

// Connect buttons to slots

connect(createButton, &QPushButton::clicked, this, &MainWindow::onCreateAccount);

connect(depositButton, &QPushButton::clicked, this, &MainWindow::onDeposit);

connect(withdrawButton, &QPushButton::clicked, this, &MainWindow::onWithdraw);

connect(searchButton, &QPushButton::clicked, this, &MainWindow::onSearchAccount);

connect(displayButton, &QPushButton::clicked, this, &MainWindow::onDisplayDetails);

connect(resetButton, &QPushButton::clicked, this, &MainWindow::onReset);

connect(exitButton, &QPushButton::clicked, this, &MainWindow::onExit);

// Connect for Enter key press

connect(amountLineEdit, &QLineEdit::returnPressed, this, &MainWindow::onDeposit);

Part 5: Error Handling & Validation

Required Validations:

  1. Account Creation:

­       Name cannot be empty

­       Account number must be positive integer

­       Account type must be selected

­       Initial deposit must be >= 0

  1. Deposit:

­       Account must exist

­       Amount must be > 0

­       Amount must be numeric

  1. Withdrawal:

­       Account must exist

­       Amount must be > 0

­       Sufficient balance required

  1. Search:

­       Account number must be entered

­       Valid account number format

Common Mistakes to Avoid

  1.  Forgetting to include Qt headers
  2.  Not using Q_OBJECT macro for slots
  3.  Mixing Qt and standard C++ types incorrectly
  4.  Not handling memory management (use new with caution)
  5.  Not connecting signals to slots properly
  6.  Forgetting to validate user input
  7.  Not updating UI after account changes
